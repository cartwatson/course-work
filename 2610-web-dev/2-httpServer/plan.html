Software Development Plan

Requirements Specification

This program will include the files from the previous static blog assignment

This program will also now feature a webserver written in python using the class "http.server — HTTP servers"

This server will direct to the previously mentioned blog pages, it will also handle the following errors: 404, 403, 418, and 301

This server will only respond to HTTP GET requests

This server will use 301 redirects in a variety of ways to make it easy to navigate for the end user; filename extensions can be ignored, filenames can be mispelled (to a degree) filenames can be similar in meaning (again, to a degree), some files will not use their filename, etc

The server will provide a favicon if the browser requests it

The server will have a debugging page that is generated at runtime

The server will handle error 418, teapot

The server will handle error 403, forbidden

System Analysis

All data for the program should come from the files provided from assign1 and the webserver

Design

CS2610Assn2.py will contain the following methods: dataHelper, do_GET

dataHelper(self, requestHandler, type_location, fileName, code)

It will take in data and handle the writing to the server

requestHandler will be a CS2610Assn2 obj

code is the error/success code that is included in the HTTP header

type_location will be the location if the code is 301 Moved Permanently

type_location will be the type if the code is not 301 Moved Permanently

filename will be a file name if code is 200 OK, otherwise it is unused

do_get(self)

do_get overrides the default function from BaseHTTPRequestHandler

it returns nothing and has no parameters besides self

it will handle GET requests

Good and bad input

if the user tries to go to a file that doesn't exist the program will give an error 404 page not found

if the user tries to go to the forbidden page it will give an error 403 forbidden

if the user tries to go to the teapot page it will give an error 418 teapot

all good input should be handled
