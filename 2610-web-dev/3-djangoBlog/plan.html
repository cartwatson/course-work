Requirements Specification

Templates must include a dynamic element

Not required to support /teapot, /forbidden, /debugging, or /bio* -> /about.html

Create two models with at least the following fields

Blog Class: title - CharField, author - CharField, content - TextField, posted - DateTimeField

Comments Class: blog - CharField, commenter - CharField, email - EmailField, content - TextField, posted - DateTimeField

Create the following dynamic pages:

Blog Home: a list view which displays the 3 most recent blogs, newest first.

Blog archive: a list view which lists all blogs, newest first.

Blog entry, a detail view which displays: An entire blog post, all comments for that blog displayed from newest to oldest, an “Add Comment” form which posts a new comment and redirects the user to back to the same Blog entry

Create a data migration to populate the database with dummy blogs and comments.

Migration must do the following:

Create at least 4 blog entries so that graders can easily verify that the Blog home and Blog archive pages operate correctly. The content in the blogs doesn't matter to me so long as they contain more than 100 words so that we may verify that you trimmed them down on the blog home and archive pages.

Each blog entry has an arbitrary title, author and posted date.

Each blog's content is arbitrary; they can all be identical.

Create at least 4 comments with the following requirements:

Create an arbitrary email address, commenter name and posted date and fill each comment with arbitrary text; they can all be identical.
